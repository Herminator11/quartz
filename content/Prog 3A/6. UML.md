## Unterschiedliche Sichten auf Software
- UML's haben 13 Diagrammtypen
- jedes schaut sich einen anderen Aspekt im System an

Diagrammtypen mit statischen Aufbau -> heißen Strukturdiagramme
	z.B. Objektdiagramme, Klassendiagramm...

Diagrammtypen mit dynamischen Aufbau -> heißen Verhaltendsdiagramme
	z.B. Aktivitätsdiagramme, Sequenzdiagramme...

- die einzelnen Elemente (sozusagen Bausteine) sind meistens mit Pfeilen verbunden
- diese Pfeile beschreiben die Beziehungen zueinander
- verschiedene Pfeile haben verschiedene Bedeutungen

## Notation von so einem Baustein:

![](static/assets/proga-10.png)

### Sichtbarkeit
- die Sichtbarkeit der einzelnen Elemente werden mit - oder + notiert
	- - steht für private (Element ist nur innerhalb der Klasse sichtbar)
	- + steht für public (auch für Instanzen anderer Klassen sichtbar)

### Klassenattribute und Klassenmethoden
- werden in UML unterstrichen

![](static/assets/proga-11.svg)

| **Symbol/Notation**                                       | **Bedeutung**                                                                                                   | **Verwendung**                                                                                                                                                                                  |
| --------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Klasse (Rechteck)**                                     | Eine Klasse wird als Rechteck dargestellt, das den Namen und eventuell Attribute und Methoden enthält.          | Zeigt eine Klasse an.                                                                                                                                                                           |
| **Attribut (Unter dem Klassennamen)**                     | Ein Attribut wird unter dem Klassennamen als Text dargestellt.                                                  | Beschreibt Eigenschaften einer Klasse, z.B. `Name`, `ID`.                                                                                                                                       |
| **Methode (Unter den Attributen)**                        | Eine Methode wird unter den Attributen als Text dargestellt.                                                    | Zeigt Methoden einer Klasse an, z.B. `berechneGehalt()`.                                                                                                                                        |
| **Assoziation (Pfeil)**                                   | Ein Pfeil, der von einer Klasse zu einer anderen zeigt, stellt eine Assoziation dar.                            | Zeigt eine Beziehung zwischen zwei Klassen an, z.B. "arbeitet mit", "gehört zu". Der Pfeil kann entweder einfach oder mit einer Raute versehen sein.                                            |
| **Vererbung (Leerer Pfeil)**                              | Ein leerer Pfeil, der zu einer übergeordneten Klasse zeigt.                                                     | Zeigt eine Vererbungsbeziehung an, d.h., eine Klasse erbt von einer anderen Klasse.                                                                                                             |
| **Implementierung (Gestrichelte Linie mit leerem Pfeil)** | Eine gestrichelte Linie mit einem leeren Pfeil zeigt, dass eine Klasse ein Interface implementiert.             | Wird verwendet, um eine Implementierung eines Interfaces durch eine Klasse darzustellen.                                                                                                        |
| **Aggregation (Raute mit Linie)**                         | Eine Aggregation wird durch eine Raute an der Seite der "besitzenden" Klasse dargestellt.                       | Zeigt eine "hat ein"-Beziehung an. Eine Klasse enthält Teile von einer anderen Klasse, aber die Teile können auch ohne die "besitzende" Klasse existieren.                                      |
| **Komposition (Ausgefüllte Raute)**                       | Eine Komposition wird durch eine ausgefüllte Raute an der Seite der "besitzenden" Klasse dargestellt.           | Eine starke Aggregation. Eine Komposition bedeutet, dass die "besitzende" Klasse die Lebensdauer der "Teile" steuert. Wenn die besitzende Klasse gelöscht wird, werden auch die Teile gelöscht. |
| **Assoziationsklasse**                                    | Eine Assoziationsklasse ist eine Klasse, die an einer Assoziation zwischen zwei Klassen beteiligt ist.          | Wird verwendet, wenn zusätzliche Attribute oder Methoden für eine Assoziation benötigt werden.                                                                                                  |
| **Multiplizität (Zahl an der Assoziation)**               | Multiplizität gibt an, wie viele Instanzen einer Klasse mit einer Instanz einer anderen Klasse assoziiert sind. | Wird an den Enden von Assoziationslinien gezeigt (z.B. 1..*, 0..1), um zu definieren, wie viele Objekte einer Klasse mit einer anderen Klasse verbunden sein können.                            |
| **Abhängigkeit (Gestrichelte Linie mit Pfeil)**           | Eine gestrichelte Linie mit einem Pfeil zeigt eine Abhängigkeit zwischen zwei Klassen an.                       | Zeigt, dass eine Klasse von einer anderen Klasse abhängt (z.B. durch Verwendung von Methoden oder Attributen).                                                                                  |
| **Interface (Kreis)**                                     | Ein Interface wird durch einen Kreis mit dem Namen des Interfaces dargestellt.                                  | Ein Interface wird durch einen Kreis dargestellt und ist eine Spezifikation von Methoden ohne Implementierung.                                                                                  |
| **Kommentar (Rechteck mit gestricheltem Rand)**           | Ein Kommentar wird durch ein Rechteck mit gestricheltem Rand dargestellt.                                       | Wird verwendet, um zusätzliche Informationen oder Erklärungen zu einem Teil des Diagramms hinzuzufügen.                                                                                         |

### Objekt Konstruktion
![](static/assets/proga-17.png)
- wenn ein Objekt am Anfang noch nicht "existiert", kann es trotzdem noch, während des Ablaufs, erstellt werden
- es wird dann einfach "weiter unten" (-> zeitlich gesehen sozusagen einfach später) gemacht/gezeichnet
- wird dann mit einem gestrichelten Pfeil mit offener Pfeilspitze eingezeichnet

### Objekt Destruktion
![](static/assets/proga-18.png)
Endet die Lebensdauer eines Objekts während des betrachteten Zeitfensters, dann endet auch die Lebenslinie in einem Kreuz.
- wenn die "Lebensdauer" eines Objekts endet (in unserem betrachteten Zeitfenster natürlich) -> endet auch die "Lebenslinie"
- das notieren wir mit einem Kreuz am Endpunkt der Linie
