# Kommunikation über Netzwerke
## Das OSI-Schichtenmodell
Beim Internet Surfen werden Daten ausgetauscht:
- Internetseite aufrufen (= fordern die Adresse an)
- laden irgendetwas herunter
- Video anschauen
Man kann das im OSI Modell darstellen.

Es gibt die (von unten nach oben):
- Bitübertragungsschicht
	- Umwandlung der Bits in ein zum Medium passendes Signal und physische Übertragung.
- Sicherungsschicht
	- Segmentierung der Pakete in Frames sowie Fehlererkennung, Fehlerbehebung und Datenflusskontrolle
- Vermittlungsschicht
	- Weiterleitung der Datenpakete zum nächsten Knoten
- Transportschicht
	- Zuordnung der Datenpakete zu einer Anwendung
- Sitzungsschicht
	- Steuerung der Verbindungen und des Datenaustauschs
- Darstellungsschicht
	- Umwandlung der systemunabhängigen Daten in ein unabhängiges Format
- Anwendungsschicht
	- Funktionen für Anwendungen sowie die Dateneingabe und -ausgabe
Erst wenn alle sieben Schichten zusammenarbeiten, haben wir eine erfolgreiche Verbindung.

> [!info]

## Netzwerkanwendungen in Java
Für die Implementierung in Java sind Klassenbibliotheken da, um die Schnittstelle zum Betriebssystem einfach nutzen zu können.

Je nach Anwendungsfall kann auf unterschiedliche Klassen aufgesetzt werden:
- Kommunikation mit Hilfe von TCP/IP -> Klasse Socket, ...
- Kommunikation mit Hilfe von HTTP(S) -> Klassen URL, URLConnection, ...

### TCP
= Transmission Control Protocol

Aufgabe:
	Datenpakete in einem dezentralisierten Netzwerk zu vermitteln und zu transportieren

![](static/assets/proga-9.png)
- Bei TCP baut ein **Servicenachfrage** (Client) eine Verbindung zu einem **Servicebringer** (Server) auf.
- Er benötigt dazu eine Adresse der Servers, die sich zusammensetzt aus der **IP-Nummer** der Suchmaschine und dem **Port**, über den der Service erreichbar ist.
- Auf der Suchmaschine könnten unterschiedliche Services parallel mit **verschiedenen Ports** existieren, nicht aber mit an einem Port.

#### Aufbau eines TCP-Segments
Datensegmente zwischen zwei Rechner bei TCP-Schicht:
- Ursprungsport und Zielport -> dienen der Adressierung des richtigen Service
- Sender vergibt **Sequenznummer** -> Empfänger stellt fest, ob Segment verloren gegangen ist
- **Acknowledgement-Nummer** -> Empfangene Segmente werden dem Sender bestätigt

#### Ports
- wird repräsentiert durch eine 16 Bit Integerzahl

### Client-Server-Programmierung
#### Server

```Java
import java.net.ServerSocket;
import java.net.Socket;

public static void main(String[] args) {
	final int port = 12345;
	try{
		ServerSocket server = new ServerSocket (port);
		Socket socket = server.accept();
		// Senden und Empfangen von Nachrichten
		socket.close();
		server.close();
	}
	catch (Exception e) {
		System.err.prinln("Error: " + e.getMessage());	
	}
}
```

Zunächst wird eine Instanz von ServerSocket mit dem Port erzeugt, auf dem gelauscht werden soll.
Der anschließende Aufruf von `accept` wartet, bis sich ein Client verbindet, und liefert eine Instanz von Socket zurück, die direkt mit dem Socket des Client verbunden ist.
Nach dem Austausch von Nachrichten müssen Socket und ServerSocket geschlossen werden.
#### Server für mehrere Clients
```java
public static void main(String[] args) {
	final int port = 12345;
	try{
		ServerSockt server = new ServerSocket (port);
		while(true) {
			Socket socket = server.accept();
			Handler handler = new Handler(socket);
			Thread thread = new Thread(handler);
			thread.start();
		}
//		server.close(); // -> unreachable code!
	}
	catch (Exception e) {
		System.err.prinln("Error: " + e.getMessage());	
	}
}
```

- man will oft nicht nur einen Client sondern mehrere bedienen
- die Kommunikation wird auf eigene **Handler-Klasse** ausgelagert und eigener Thread gestartet
	- dieser übernimmt Kommunikation mit neu verbundenem Client

- wenn im try-Block Ressourcen allokiert und wieder freigegeben werden müssen ist `close`schwierig
- hier können Ressourcen unmittelbar hinter try angefordert werden
	- werden dann automatisch freigegeben, wenn try-Block verlassen wird
	- close ist nicht mehr notwendig

#### Client
```java
public static void main (String[] args) {
	InetSocketAddress address = 
	new Inet SocketAddress("localhost", 12345);

	try (Socket socket = new Socket()) {
		socket.connect(address);
		//mit socket: Senden und Empfangen
	}
	catch (Exception e){...}
}
```
- Client stellt Kontakt zum Server her -> braucht Adresse des Services (Rechner + Port)
- Socket wird erzeugt und mittels connect Verbindung hergestellt
	- Voraussetzung: An der angesprochenen Adresse wartet ein ServerSocket mit accept auf Kontaktaufnahmen
