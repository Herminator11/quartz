## Joins
- kombinieren von Daten aus zwei (oder mehr) Tabellen
### Inner Joins
![](static/assets/inner-join-example.webp)
<u>Natural Inner Join</u>
- Definition: Kombiniert Zeilen aus zwei Tabellen basierend auf allen Spalten mit demselben Namen und Datentyp.
- doppelt benannte Namen werden mit Projektion "herausprojeziert"
- am Schluss kommt kein Attribut mehrfach auf

<u>Inner (Theta) Join</u>
-> allg. Join
- die condition ist hier schon gegeben
- keine Projektion findet hier statt

### Outer Joins
- kombinieren Zeilen aus zwei Tabellen
- behalten Zeilen, die keine Übereinstimmung in der anderen Tabelle haben
- -> Fehlende Werte werden mit `NULL` gefüllt

<u>Left Outer Join</u>
- Beinhaltet alle Zeilen aus der linken Tabelle und die passenden Zeilen aus der rechten Tabelle.
- Wenn keine Übereinstimmung -> `NULL` für Spalten der rechten Tabelle

```sql
SELECT *
FROM TabelleA
LEFT OUTER JOIN TabelleB
ON TabelleA.Spaltenschluessel = TabelleB.Spaltenschluessel;
```

<u>Right Outer Join</u>
- Beinhaltet alle Zeilen aus der rechten Tabelle und die passenden Zeilen aus der linken Tabelle
- Fehlt Übereinstimmung -> `NULL` für die Spalten der linken Tabelle

```sql
SELECT *
FROM TabelleA
RIGHT OUTER JOIN TabelleB
ON TabelleA.Spaltenschluessel = TabelleB.Spaltenschluessel;
```

<u>Full Outer Join</u>
- Kombiniert Ergebnisse von `LEFT OUTER JOIN` und `RIGHT OUTER JOIN`
- enthält alle Zeilen aus beiden Tabellen
- Fehlende Werte in einer Tabelle werden durch `NULL` ersetzt

```sql
SELECT *
FROM TabelleA
FULL OUTER JOIN TabelleB
ON TabelleA.Spaltenschluessel = TabelleB.Spaltenschluessel;
```

### Unterschiede & Beispiele

| Join-Typ         | Zeilen aus Tabelle A | Zeilen aus Tabelle B | Nicht passende Zeilen |
| ---------------- | -------------------- | -------------------- | --------------------- |
| Natural Join     | Nur übereinstimmende | Nur übereinstimmende | Keine                 |
| Inner Join       | Nur übereinstimmende | Nur übereinstimmende | Keine                 |
| Left Outer Join  | Alle                 | Übereinstimmende     | Links behalten        |
| Right Outer Join | Alle                 | Alle                 | Rechts behalten       |
| Full Outer Join  | Alle                 | Alle                 | Beide behalten        |

**Beispieldaten:**
TabelleA: Kunden

| KundenID | Name  |
| -------- | ----- |
| 1        | Alice |
| 2        | Bob   |
| 3        | Carol |

TabelleB: Bestellungen

| BestellungID | KundenID |
| ------------ | -------- |
| 101          | 1        |
| 102          | 2        |
| 103          | 4        |
Natural Inner Join:

```sql
SELECT *
FROM Kunden
NATURAL JOIN Bestellungen;
```
Ergebnis: Nur Alice und Bob (übereinstimmende KundenID)

Inner Theta Join:

```sql
SELECT Kunden.Name, Bestellungen.BestellungID
FROM Kunden
INNER JOIN Bestellungen
ON Kunden.KundenID = Bestellungen.KundenID;
```

Left Outer Join:

```sql
SELECT Kunden.Name, Bestellungen.BestellungID
FROM Kunden
LEFT OUTER JOIN Bestellungen
ON Kunden.KundenID = Bestellungen.KundenID;
```
Ergebnis: Alice, Bob und Carol (ohne Bestellung -> BestellungID = Null)

Full Outer Join:

```sql
SELECT *
FROM Kunden
FULL OUTER JOIN Bestellungen
ON Kunden.KundenID = Bestellungen.KundenID;
```
Ergebnis: Alice, Bob, Carol und Bestellung 103 (kein passender Kunde)


---
NATURAL JOIN kombiniert Tabellen basierend auf gleichnamigen Spalten.
Diese Spalten werden in der Ergebnistabelle nur einmal angezeigt.
