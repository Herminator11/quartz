## 2.1 Definition des relationalen Modells
### 2.1.1 Mathematischer Formalismus

geg.: n nicht - notwendigerweise unterschiedliche Wertebereiche (Domänen):
	D<sub>1</sub>, D<sub>2</sub>,..., D<sub>n</sub>; D<sub>i</sub> = D<sub>j</sub> zulässig für i ≠ j

Domänen dürfen nur atomare Werte enthalten
d.h.: nicht strukturierbar 
→ Beispiele für <u>gültige</u> Domänen: 
	Zahlen (N, Z, Q, etc.), 
	Zeichenketten (Ausnahme obwohl zerlegbar → haben eigentlich eine Struktur)

→ Beispiele für <u>ungültige Domänen</u>: 
	(Mengen von) Tupeln,
	Mengen

Relation R ist definiert als die Teilmenge des Kartesischen Produkt (Kreuzprodukts) der n Domänen:

Ein Element der Menge R heißt n-stelliges Tupel

## P1. (einfaches) SQL SELECT Statement
### P1.1 Structured English QUEry Language (SEQUEL / SQL)

- (praktische) Anfragesprache
- basiert auf relationaler Algebra (bietet div. Erweiterungen!)
- standardisiert
- wird von allen rel. DBS implementiert
- hier: Fokus Oracle-SQL

### P1.2 Syntax

SELECT * | C<sub>1</sub>, C<sub>2</sub>, ..., C<sub>n</sub>
FROM T;
→ Schlüsselwörter immer groß halten → Lesbarkeit!

z.B.: 
- SELECT * FROM Professoren;
→ liefert alles / alle Tabellenspalten zurück 

- SELECT Name, Rang FROM Professoren;
→ liefert Name und Rang Spalten zurück

### P1.3 Arithmetische Operatoren

z.B.:
SELECT Nachname, (Gehalt * 12) + 2000
FROM Mitarbeiter;
→ man kann mit Werten in den Statements rechnen
→ falls in bspw. Gehalt nichts drinsteht (null!) dann ist gesamte Aussage <mark style="background: #FF5582A6;">null</mark>

### P1.4 Alternative Spaltennamen

z.B.:
SELECT (Gehalt * 12) + 2000 AS Jahresgehalt FROM Mitarbeiter;
→ ändert den Namen der Formel in einen Namen → hier Jahresgehalt

alternativ für mehr-wörtige Benennung
SELECT (Gehalt * 12) + 2000 "Gehalt pro Jahr" FROM Mitarbeiter;

### P1.5 Konkatenation und String-Literale
z.B.: 
SELECT Vorname || ' ' || Nachname || ' arbeitet als ' || Funktion AS "Wer arbeitet als was?"
FROM Mitarbeiter;
→ liefert Spalte mit Vorname Nachname und unseren Strings

### P1.6 Duplikate vermeiden

SELECT RANG
FROM PROFESSOREN;
→ liefert Rang von Professoren
→ W2 doppelt (Duplikat) wird nicht automatisch eliminiert

SELECT <mark style="background: #FF5582A6;">DISTINCT</mark> Rang FROM Professoren;
→ eliminiert Duplikate
