- Computer kann nur 0 und 1 verarbeiten (an und aus)
- mit Binärsystem lassen sich auch größere Zahlen darstellen
- aber was ist mit negativen Zahlen

> [!info] Kurz erklärt
> Zweierkomplement ist eine Darstellungsform für binäre Zahlen, mit dem man positive und negative Zahlen abbildet.


## Zweierkomplementdarstellung

Um negative und positive Zahlen im Binärcode abzubilden, benutzt man ein Bit als Vorzeichen.
Zwei Verfahren:
<u>Least Significant Bit (LSB)</u> - Verfahren
- Bit 0 (also das Bit ganz rechts) wird als Vorzeichen genutzt

<u>Most Significant Bit (MSB)</u> - Verfahren
- das Bit was ganz links steht wird als Vorzeichenbit genutzt
- dabei steht eine 0 an der MSB-Stelle für positive Zahlen
- und eine 1 für negative Zahlen

Im 2er Komplement kann man einen Wertebereich von
<center>-2<sup>n-1</sup>, ..., 0, ..., 2<sup>n-1</sup> - 1</center>
darstellen.
In dieser Formel steht n für die Anzahl der Bits. Für 4 Bits kannst du also die Zahlen
<center>1000<sub>2</sub> = -8<sub>10</sub>&nbsp bis &nbsp0111<sub>2</sub>&nbsp = 7<sub>10</sub></center>
abbilden.

## Zweierkomplement bilden
Eine Binärzahl wird ins 2er Komplement umgewandelt, indem die einzelnen Bits erst invertiert und dann eine 1 hinzuaddiert wird.

Bsp.: Zahl -5 im 2er Komplement darstellen.
1. Betrag der Zahl in Binärcode
	 → 5 = 0101
2. Invertieren der einzelnen Stellen der Binären Zahl

*Invertieren bedeutet, dass jede 0 mit einer 1 und jede 1 mit einer 0 vertauscht wird.*
	 0101 → 1010

3. 1 muss noch addiert werden
	 → 1010 + 0001 = 1011 = -5 (dezimal)

## Zweierkomplement umwandeln
- vor Umwandlung erst MSB anschauen
- ist MSB eine 0 kann die Zahl direkt umgerechnet werden
- ist MSB eine 1 muss Binärcode erst invertiert werden, 1 dazu addieren und dann erst ins Dezimalsystem umgewandelt werden

Bsp.: 1011
- MSB ist 1 also invertieren
	- 1011 = 0100
- daraufhin eine 1 addieren
	- 0100 + 0001 = 0101 = 5 (dezimal)

- Jetzt muss man sich das negative Vorzeichen dazu denken und man erhält die -5

### Trick:
Die Umwandlung ins Dezimalsystem geht auch schneller.
Dazu rechnet man die Binärzahl wie gewohnt in das Dezimalsystem um, jedoch subtrahiert das MSB, anstatt es zu addieren.

Also:
1011 = -8 • 1 + 4 • 0 + 2 • 1 + 1 • 1 = -5 (dezimal)
