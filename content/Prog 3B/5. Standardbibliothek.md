- Header-Dateien der Standardbibliothek werden in der Regel ohne die Dateiendung `.h`eingebunden, z.B.

```cpp
#inlcude <iostream> // Ein- und Ausgabe
#include <vector> // Dynamische Arrays
# include <string> // Zeichenketter
```

## 5.1 Container
- Container sind Klassen, die eine Sammlung von Objekten verwalten
- unterscheiden sich in der Art, wie die Daten gespeichert werden und wie auf die Daten zugegriffen werden kann.
- verfügen über eine einheitliche Schnittstelle
	-> Container können unabhängig von ihrer konkreten Implementierung verwendet werden

<u>Wichtige Container-Klassen</u>:
- `std::vector`, 
- `std::list`,
- `std::map`

- in Container können beliebige Datentypen gespeichert werden
- Datentypen <span style="background:#d4b106">müssen</span> kopierbar sein
	-> d.h. sie müssen einen [[3. Klassen#3.4.6 Der Kopierkonstruktor|Kopierkonstruktor]] und einen Zuweisungsoperator bereitstellen
- um zu definieren, welcher Datentyp in einem Container gespeichert werden soll, wird der Datentyp in spitzen Klammern angegeben, z.B.:
	`std::vector<int>`, `std::list<std::string>` usw.

> [!info]
> Container haben im Vergleich zu klassichen `C`-Arrays viele Vorteile: automatische Speicherverwaltung, dynamische Größe, einheitliche Schnittstellen etc. Verwenden Sie in `C++` daher immer Container anstelle Arrays.

### 5.1.1 'std::vector'
- `std::vector` ist ein dynamisches Array
- speichert Elemente in einem zusammenhängenden Speicherbereich
- Zugriff auf Elemente erfolgt über den Index
- `std::vector` kann 
	- Elemente am Ende hinzufügen oder entfernen
	- auf beliebig viele Elemente zugreifen
	- Elemente an beliebiger Stelle einfügen


```cpp
#include <vector>

int main()
{
    // Ein leeres Array erstellen
    std::vector<int> v;

    // Elemente hinzufügen
    v.push_back(1);
    v.push_back(2);
    v.push_back(3);

    // Letztes Element entfernen
    v.pop_back();

    // Elemente ausgeben
    for (int i = 0; i < v.size(); ++i)
    {
        std::cout << v[i] << std::endl;
    }

    return 0;
}
```

- `std::vector` verwaltet automatisch im Hintergrund den Speicher für die Elemente
- wenn `std::vector` zu klein wird, wird mehr Speicher reserviert
	-> Elemente werden in den neuen Speicherbereich kopiert
	-> Kopieren kostet Zeit!
- wenn bewusst, wie viele Elemente gespeichert werden müssen, kann man Größe des Vektors bei Erstellung festlegen
	-> `std::vector<int> v(10);` erstellt einen Vektor mit 10 Elementen

> [!note]
> Ein Vektor ist ein guter Container, wenn Sie eine dynamische Liste von Elementen benötigen, auf die Sie mit einem Index zugreifen möchten.

### 5.1.2 'std::list'
- `std::list` ist eine doppelt verkettete Liste

zu Verketteten Listen: https://tinyurl.com/26glr8yt
![](static/assets/progb-3.png)

- speichert Elemente in beliebiger Reihenfolge
- Zugriff erfolgt über Iteratoren.
- direkter Zugriff auf ein bestimmtes Element über den Index ist nicht möglich.

- Ein `std::list` kann Elemente am Anfang, am Ende oder an beliebiger Stelle einfügen oder entfernen.

```cpp
#include <list>

int main()
{
    // Eine leere Liste erstellen
    std::list<int> l;

    // Elemente hinzufügen
    l.push_back(1);
    l.push_back(2);
    l.push_back(3);

    // Erstes Element entfernen
    l.pop_front();
    // Letztes Element entfernen
    l.pop_back();

    // Elemente ausgeben
    for (auto it = l.begin(); it != l.end(); ++it)
    {
        std::cout << *it << std::endl;
    }

    return 0;
}
```

- ähnliches Interface zu `std::vector` 
- nur der Zugriff auf Elemente unterscheidet sich.

> [!note]
> Eine Liste ist ein guter Container, wenn Sie häufig Elemente einfügen oder entfernen müssen.

### 5.1.3 Iteratoren
- sind Objekte, die den Zugriff auf die Elemente eines Containers ermöglichen
- ein Iterator zeigt auf ein bestimmtes Element im Container
- kann verwendet werden, um auf das Element zuzugreifen oder das Element zu verändern.
- sie können inkrementiert und dekrementiert werden, um auf das nächste bzw. vorherige Element zu zeigen.

- im Beispiel der Liste oben wird ein Iterator `it` verwendet, um über die Elemente der Liste zu iterieren.
- Iterator wird mit `l.begin()` auf das erste Element gesetzt
- dann wird er in einer Schleife solange inkrementiert, bis er auf das Element `l.end()` zeigt, das hinter dem letzten Element steht.

- Zugriff auf das aktuelle Element erfolgt über den Dereferenzierungsoperator `*it`.
- Typ des Iterators hängt vom Container ab, auf den er zeigt
- in obigem Beispiel ist der Typ von `it` -> `std::list<int>::iterator`.
- Compiler kann den Typ des Iterators auch automatisch ableiten, indem auto verwendet wird.

<u>Noch kürzer:</u>
- `range-based for`-Schleife:

```cpp
for (const int& i : l)
{
    std::cout << i << std::endl;
}
```
- hier erzeugt Compiler automatisch einen Iterator, der über alle Elemente der Liste iteriert.
- der Wert auf den den Iterator zeigt, wird in `i` gespeichert

> [!note]
> Iteratoren sind ein mächtiges Konzept in `C++`, das es ermöglicht, Container unabhängig von ihrer konkreten Implementierung zu verwenden. Egal welchen Container Sie nutzen, Sie können immer mit Iteratoren auf die Elemente zugreifen.

### 5.1.4 'std::map'
- `std::map` ist eine assoziative Datenstruktur
- speichert Schlüssel-Wert-Paare
- man kann definieren, welche Datentypen für den Schlüssel und den Wert verwendet werden sollen
	-> z.B. `std::map<std::string, int>` für ein string-Schlüssel und integer-Werte.
- Zugriff auf Elemente erfolgt über den Schlüssel (anstelle über den Index)
	-> wie bei einem Array, mit `map["schluessel"]`.
- Schlüssel müssen in der Map eindeutig sein.

- Beim Iterieren über die Elemente bekommt man Paare von Schlüssel und Wert als Objekt vom Typ `std::pair`
- Schlüssel ist dabei `pair.first` und der Wert `pair.second`

```cpp
#include <map>

int main()
{
    // Eine leere Map erstellen
    std::map<std::string, int> m;

    // Elemente hinzufügen
    m["one"] = 1;
    m["two"] = 2;
    m["three"] = 3;

    // Erstes Element ausgeben
    std::cout << m["one"] << std::endl;

    // Elemente ausgeben
    for (const auto& p : m)
    {
        std::cout << p.first << ": " << p.second << std::endl;
    }

    return 0;
}
```

> [!note]
> Eine Map ist ein guter Container, wenn Sie Schlüssel-Wert-Paare speichern möchten und auf die Elemente über den Schlüssel zugreifen möchten.

## 5.2 Algorithmen
- arbeiten meist mit Sequenzen von Elementen, weshalb sie mit Containern nutzbar sind.
- Beispiele sind:
	- `std::sort`
	- `std::find`

- Sequenz, auf der ein Algorithmus arbeitet, wird durch zwei Iteratoren definiert
	- geben Anfang und das Ende der Sequenz an
	-> können deshalb mit allen Containern verwendet werden, die Iteratoren unterstützen.

- Algorithmen sind in der Header-Datei `<algorithm>` definiert

### 5.2.1 'std::sort'
- `std::sort` sortiert Elemente einer Sequenz.
- Algorithmus arbeitet *in-place*
	-> d.h. er sortiert direkt die angegebene Sequenz und verwendet den `Quicksort`-Algorithmus (`O(n log n))`

```cpp
#include <algorithm>
#include <vector>

int main()
{
    std::vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5};

    // Vektor sortieren
    std::sort(v.begin(), v.end());

    // Elemente ausgeben
    for (const auto& i : v)
    {
        std::cout << i << std::endl;
    }

    return 0;
}
```

- `std::sort` funktioniert mit allen Datentypen, für die eine *kleiner-als-Relation* definiert ist
	-> d.h. bei denen definiert ist, wie zwei Elemente des Datentyp verglichen werden können
- Für eingebauten Datentypen wie `int`, `float`, `double` etc. ist das bereits definiert
- Für einige Datentypen muss man ggf. einen Vergleichsoperator definieren

<u>Alternativ</u>:
- man kann eine eigene Vergleichsfunktion angeben, wenn man eine spezielle Sortierreihenfolge benötigt:

```cpp
#include <algorithm>

// Vergleichsfunktion: Nimmt zwei Elemente und gibt zurück,
// ob das erste Element größer ist als das zweite
bool compare(int a, int b)
{
    return a > b;
}

int main()
{
    std::vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5};

    // Vektor sortieren mit eigener Vergleichsfunktion
    std::sort(v.begin(), v.end(), compare);

    ...

    return 0;
}
```

### 5.2.2 'std::find'
- `std::find` sucht ein Element in einer Sequenz
- Algorithmus gibt einen Iterator zurück
	-> zeigt auf das erste Vorkommen des gesuchten Elements
- wenn das Element nicht gefunden wird, wir der Iterator `end()` zurückgegeben.

```cpp
#include <algorithm>

int main()
{
    std::vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5};

    // Element suchen
    auto it = std::find(v.begin(), v.end(), 4);

    if (it != v.end())
    {
        std::cout << "Element gefunden: " << *it << std::endl;
    }
    else
    {
        std::cout << "Element nicht gefunden" << std::endl;
    }

    return 0;
}
```
