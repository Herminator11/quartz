## 7.1 Was sind Templates?
- generische Funktionen oder Klassen, die mit verschiedenen Datentypen oder Konstanten arbeiten können
- sind in der Lage, Code zu generieren, der auf verschiedenen Datentypen basiert
=> ermöglicht Funktionen und Klassen zu schreiben, die mit beliebigen Datentypen arbeiten können, ohne dass Code für jeden Datentyp neu geschrieben werden muss

"Das Template dient also als Bauplan bzw. Schablone für die Erstellung von Funktionen oder Klassen."

Beispiel:

```cpp
class Point2D {
private:
    int x;
    int y;
public:
    Point2D(int x, int y) : x(x), y(y) {}
    int getX() const { return x; }
    int getY() const { return y; }
};
```
- Klasse `Point2D` repräsentiert eine 2D-Koordinate
- Typ der Koordinaten ist `int`
Wir möchten aber eine Klasse, die mit `double`-Koordinaten arbeitet:
- wir müssten eine neue Klasse `Point2DDouble` schreiben
- würde zwar genauso aussehen wie `Point2D`, aber mit `double`
-> nicht sehr effizient

## 7.2 Template - Klassen
Da wir mit Templates eine generische Klasse schreiben können, die mit verschiedenen Datentypen arbeiten kann, könnte unsere `Point2D`-Klasse so aussehen:

```cpp
template <typename T>
class Point2D {
private:
	T x;
	T y;
public:
	Point2D(T x, T y) : x(x), y(y) {}
	T getX() const {return x; }
	T getY() const {return y; }
};
```
- `template` vor Klassendeklaration, um eine Template-Klasse zu erstellen
- Ausdruck `<typename T>` definiert einen Typparameter `T`, der als Platzhalter für den tatsächlichen Datentyp verwendet wird
	-> `T` kann hier durch jeden beliebigen Datentyp ersetzt werden
	-> Klasse `Point2D` kann also mit `int`,`double`,`float` etc. verwendet werden
- Es können auch mehrere Typparameter definiert werden, z.B.
	- `<typename T1, typename T2>`.

- Um eine Instanz der Klasse `Point2D` zu erstellen, müssen wir den Datentyp angeben, den wir verwenden möchten. z.B.:

```cpp
Point2D<int> point1(1, 2);
Point2D<double> point2(1.5, 2.5);
```
- Compiler generiert Code für die Klasseninstanz `Point2D<int>` und `Point2D<double>`
- er ersetzt `T` durch angegebenen Datentyp und kompiliert entstandenen Code
- Compiler erstellt für jede konkrete Typisierung eine neue Klasse
	-> Klassen `Point2D<int>` und `Point2D<double>` sind zwei verschiedene Klassen
	-> haben nichts miteinander zu tun, obwohl sie auf selben Template basieren
	-> Es können z.B. keine `Point2D<int>`-Objekte in eine `Point2D<double>`-Variable kopiert werden.

- Template-Klassen werden erst bei Verwendung mit Datentyp kompiliert
	-> deswegen werden sie komplett in Header-Dateien definiert
	-> stellt sicher, dass Compiler die komplette Definition sehen kann, wenn sie benötigt wird.

## 7.3 Template-Funktionen
- sind Funktionen, die mit verschiedenen Datentypen oder Konstanten arbeiten können
- ähnliche Syntax wie bei Template-Klassen
	- nur Typparameter dient als Platzhalter für den Datentyp der Funktion

Beispiel:

```cpp
template <typename T>
T add(T a, T b) {
	return a + b;
}

int main() {
	int sum1 = add(1, 2);
	double sum2 = add (1.5, 2.5);
	return 0;
}
```
