## 8.1 Exceptions
"*Exceptions sind ein Mechanismus, um Fehlerbehandlung zu realisieren. Sie ermöglichen es, Fehlerbedingungen zu signalisieren und in einem anderen Teil des Programms zu behandeln.*"

### 8.1.1 Exceptions in C++
- werden mit `throw` geworfen
- mit `try` und `catch` behandelt

- `throw`-Statement erzeugt ein Objekt, das den Fehlerzustand beschreibt
- dieses wird dann weitergereicht, bis es von einem `catch`-Block abgefangen wird.

```cpp
#include <iostream>
#include <stdexcept>

void foo() {
    throw std::runtime_error("Fehler in foo");
}

void bar() {
    foo();
}

int main() {
    try {
        bar();
    } catch (const std::runtime_error& e) {
        std::cerr << "Fehler: " << e.what() << std::endl;
    }
}
```
- hier wird in `foo` eine `std::runtime_error`-Exception geworfen.
- Diese wird nicht in `bar` behandelt, sondern in `main`
- Dort wird eine Fehlermeldung ausgegeben
- Typ des Exceptions-Objekts bestimmt, welcher `catch`-Block verwendet wird
	-> hier alle `std::runtime_error`-Exceptions

- `std::exception` ist die Basisklasse für alle Exceptions in C++
- definiert eine Methode `what()`
	-> gibt eine Beschreibung des Fehlers zurück

Im Header `stdexcept` sind weitere Standard-Exception-Klassen definiert, die von `std::exception` erben:

```cpp
namespace std {
    class logic_error;
    class domain_error;
    class invalid_argument;
    class length_error;
    class out_of_range;
    class runtime_error;
    class range_error;
    class overflow_error;
    class underflow_error;
}
```
=> macht Sinn die zu verwenden, die am besten zum Fehler passen
