## 2.1 Strukturen
- einfacher Datencontainer
- kann auch Funktionen beinhalten

```cpp
struct Point {
	int x;
	int y;

	void print() {
		std::cout << "(" << x << ", " << y << ")" << std::endl;
		}
};
```

Deklaration der Struktur in C++:

```cpp
int main() {
	Point p;

	p.x = 1;
	p.y = 2;

	p.print();

	return 0;
}
```
- Zuerst wird ein Objekt `p`als Instanz der Struktur `Point`erstellt.
- Anschließend werden die Werte `x`und `y`des Punktes gesetzt
- Zum Aufrufen von Funktionen innerhalb einer Struktur wird der Punkt-Operator `.`verwendet.
- Mit `p.print()`wird die Methode `print`des Punktes p aufgerufen.

> [!info]
> Funktionen innerhalb einer Struktur bzw. Klasse werden als **Methoden** bezeichnet.

## 2.2 Ein- und Ausgabe
- Für die Ausgabe in `C++`gibt es `std::cout`

- `std::cout`ist keine Funktion, sondern ein s.g. Stream-Objekt.
	-> Aus diesem Grund verwendet man auch nicht etwa `std::cout("text")`, sondern den Stream-Operator `<<`.
- Ein Stream ist eine Sequenz von Bytes, die von einer Quelle zu einem Ziel fließen.
- Es gibt in `C++`verschiedene Arten von Streams
- `cout`ist der Standardausgabestream, der auf die Konsole schreibt.

- Operator `<<`wird verwendet, um Daten in den Stream zu schreiben.
- In welche Richtung die Daten fließen, lässt sich anhand des Operators intuitiv erkennen.

Bsp.:
```cpp
std::cout << "text" << std::endl;
```
- Daten fließen von rechts nach links
- also zuerst vom String "text" in den Stream `std::cout`
- Anschließend wird ein Zeilenumbruch (`std::endl`) in den Stream geschrieben.

- Für Eingabe gibt es den Stream `std::cin`
- Operator `>>`wird verwendet, um Daten aus dem Stream zu lesen.

Bsp.:
```cpp
int x;
std::cin >> x;
```
- Daten fließen von `std::cin`in Variable `x`.

## 2.3 Referenzen
In `C`gibt es Zeiger, um auf Speicheradressen zu verweisen. Ein Anwendungsfalls dafür ist *call-by-reference*, wobei eine Funktion eine Referenz auf eine Variable erhält und somit den Wert der Variablen ändern kann.

Problem:
Bei Verwendung von Zeigern kann es leicht zu Fehlern kommen. Zum Beispiel kann ein Zeiger auf eine ungültige Speicheradresse zeigen, was meist zum Absturz des Programms führt und unbedingt vermieden werden muss!

In  <font color="#8064a2">C++</font> gibt es daher s.g. **Referenzen**!
- sichere Alternative zu Zeigern.
- Verhalten sich wie Zeiger, aber können nicht auf `NULL`zeigen
- muss bei der Initialisierung auf ein konkretes Objekt bzw. eine Variable verweisen.

- Eine Referenz wird mit dem `&`-Operator definiert, also z.B. `Point &a` anstatt `Point *a`.

Beispiel eine Funktion `swap()`:
- tauscht die Werte zweier Punkte aus
```cpp
void swap(Point &a, Point &b) {
	Point tmp = a;
	a = b;
	b = tmp;
}

int main() {
	Point p1 = {1, 2};
	Point p2 = {3, 4};

	swap(p1, p2); //Kein &-Operator notwendig!

	return 0;
}
```

> [!info]
> Nutzen Sie für *call-by-reference in `C++`Referenzen statt Zeigern*.

## 2.4 Namespaces
- werden verwendet, um den Gültigkeitsbereich von Funktionen, Klassen und Variablen zu definieren.
- Namespace ist ein Bereich in dem Namen eindeutig sein müssen.
	-> zwei Funktionen mit gleichen Namen können in unterschiedlichen Namespaces existieren, ohne dass es zu Konflikten kommt.

- wird mit dem `namespace`-Schlüsselwort definiert
- der Name des Namespaces wird in geschweiften Klammern angegeben.
- Innerhalb des Namespaces können Funktionen, Klassen und Variablen definiert werden.

Beispiel `std::cout`:
- `std`ist ein Namespace, der Funktionen und Klassen der C++ - Standardbibliothek beinhaltet.

- auch eigene Namespaces kann man definieren
Beispiel:
- Namespace `geometry` der u.a. die Struktur `Point` enthält:
```cpp
namespace geometry {
	struct Point {
		…
	};
}
```
- hier müssen wir auf Struktur `Point` mit `geometry::Point` zugreifen.

> [!note]
> Um den Code übersichtlicher zu gestalten, können Sie auch den `using namespace` - Befehl verwenden. Zum Beispiel sieht man häufig `using namespace std;` am Anfang einer cpp-Datei. Damit kann man dann wieder direkt (also ohne `std::…`) auf die Funktionen und Klassen des `std`-Namespaces zugreifen. Nutzen Sie dies jedoch nur in cpp-Dateien und nicht in Headerdateien, um Namenskonflikte beim inkludieren zu vermeiden!

## 2.5 Der Datentyp 'string'
- in `C` gibt es keinen Datentyp für Zeichenketten.
- in `C++` gibt es den Datentyp `std::string`, der in der Headerdatei `string`definiert ist.
- Ein `std::string` ist ein Objekt, das eine Zeichenkette repräsentiert.
- Mittels des '<font color="#8064a2">=</font>' -Operators können Zeichenketten einem String zugewiesen werden

```cpp
#include <iostream>
#include <string>

int main() {
    std::string name = "Alice";

    std::cout << "Hello, " << name << "!" << std::endl;

    return 0;
}
```
> [!info]
> Nutzen Sie in `C++` den Datentyp `std::string` für Zeichenketten.

### 2.5.1 stringstream
- um verschiedene Datentypen in einen String zu konvertieren, kann man `std::stringstream` verwenden
	-> im Header `sstream` definiert
- `std::stringstream` ist ein Stream-Objekt, das Daten in einen String schreibt
- man kann verschiedene Datentypen in den Stream schreiben und anschließend den gesamten Inhalt des Streams als String auslesen
```cpp
#include <iostream>
#include <string>
#include <sstream>

int main() {
	std::stringsteam ss;

	int x = 42;

	double y = 3.14;

	ss << "x = " << x << ", y = " << y;

	std::string result = ss.str();

	std::cout << result << std::endl;

	return 0;
}
```
- `ss` ein Objekt / eine Instanz von `std::stringstream`
- `str()` gibt den gesamten String aus dem Stream als fertigen `std::string` zurück
