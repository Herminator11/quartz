> [!info]
> Eine Klasse modelliert häufig einen realen Gegenstand oder eine abstrakte Entität bzw. einen abstraken Akteur. In ihr werden die Eigenschaften und das Verhalten der Entität beschrieben und gebündelt.

In `C++` werden Klassen mit dem Schlüsselwort `class` deklariert:
```cpp
class Vector2D {
	//Attribute
	int x;
	int y;

	//Methoden
	void print();
	void add(const Vector 2D& otherVector);
	…
};
```
- Innerhalb der Klasse definieren wir Attribute
	-> also alle Eigenschaften bzw. Variablen, die ein Objekt dieser Klasse haben kann.
- wir deklarieren/definieren Methoden (also Funktionen), die das Verhalten der Klasse beschreiben.

> [!info]
> Attribute und Methoden, die innerhalb einer Klasse definiert sind, werden auch als _Member_ bezeichnet.

## 3.1 Deklaration vs. Definition

Deklaration:
- gibt an, wie die Funktion aufgerufen werden kann
- also wie viele und welche Parameter sie erwartet und welchen Rückgabewert sie hat

Definition:
- gibt an, was die Funktion genau macht, also den Funktionsrumpf 
	-> das, was innerhalb der geschweiften Klammern der Funktion steht

> [!info]
> Versuchen Sie, eine Funktion aufzurufen, die nur deklariert, aber nicht definiert ist, kommt es zu einem **Linker-Fehler**: Der Linker kann die Funktion nicht finden und bricht den Kompiliervorgang ab.

Im obigen Beispiel sind bisher nur die Methoden der Klasse *deklariert*.
Für die Definition muss man noch den Funktionsrumpf schreiben (-> man sagt auch *implementieren*).
In `cpp` gibt es verschiedene Möglichkeiten, wo man die Methoden einer Klasse definieren kann.
Hier erstmal Definition innerhalb der Klasse:
- dazu schreibt man den Funktionsrumpf direkt unter die Deklaration der Methode
```cpp
class Vector2D {
    // Attribute
    int x;
    int y;

    // Methoden
    void print() {
        std::cout << "(" << x << ", " << y << ")" << std::endl;
    }

    void add(const Vector2D& otherVector) {
        x += otherVector.x;
        y += otherVector.y;
    }
    ...
};
```

## 3.2 Sichtbarkeit und Zugriff
- Bei Deklaration der Attribute und Methoden muss man Sichtbarkeit angeben
	-> *public* oder *private*

- Standardmäßig sind alle Attribute und Methoden einer Klasse **private**
	-> d.h. sie sind nur innerhalb der Klasse sichtbar und können nur von Methoden der Klasse selbst genutzt werden.

- Um auf Attribute und Methoden einer Klasse von außen zuzugreifen, müssen wir diese explizit als **public** deklarieren
- Dies geschieht mit Schlüsselwörtern `public:` und `private:` vor den jeweiligen Membern:
```cpp
class Vector2D {
private:
    int x;
    int y;

public:
    void print() {...};
    void add(const Vector2D& otherVector) {...};
    ...
};
```

- es ist empfehlenswert, die Attribute einer Klasse als **private** zu deklarieren.
	-> Abstraktion und Datenkapselung
	- sind meist interne Details der Klasse, die von außen nicht direkt verändert werden sollten
	 - Ein Zugriff auf die sollte über *Getter* und *Setter* erfolgen.

Im Beispiel könnte man der Klasse `Vector2D` Getter- und Setter-Methoden für Attribute `x` und `y` hinzufügen:
```cpp
class Vector2D {
private:
	int x;
	int y;

public:
	int getX() const {
		return x;
	}
	int getY() const {
		return y;
	}
	void setX(int newX) {
		x = newX;
	}
	void setY(int newY) {
		y = newY;
	}
	...
}
```
- `const` nach den Methoden deklariert, dass die Methode keine Attribute der Klasse verändern darf

> [!info]
> Attribute sollten immer **private** deklariert werden! Methoden hingegen können **public** oder **private** sein, je nachdem, ob sie von außen aufgerufen werden sollen oder nicht.

## 3.3 Objekte
- konkrete Instanz einer Klasse
	-> Objekt hat also die Eigenschaften und das Verhalten, das in der Klasse definiert ist.
- In `C++` erzeugen wir Objekte einer Klasse, indem wir eine Variable vom Typ der klasse deklarieren:

```cpp
int main() {
    Vector2D v1;
    v1.setX(3);
    v1.setY(4);

    Vector2D v2;
    v2.setX(1);
    v2.setY(2);

    v1.add(v2);
    v1.print();
    ...
}
```
- mit dem `.`-Operator kann man die Attribute und Methoden eines Objekts zugreifen (wenn es die Sichtbarkeit zulässt)
- im Beispiel nutzen wir Setter, um beiden Objekten unterschiedliche Werte zu geben und addieren den Vektor `v2` auf `v1` und geben dann das Ergebnis mit `print()` aus.

## 3.4 Konstruktoren
- In obigem Beispiel haben wir die Attribute der Klasse `Vector2D` nicht initialisiert.
- man könnte direkt bei der Deklaration der Attribute die initialen Werte angeben

- Aber in `C++` nutzen wir den *Konstruktor*, um das Objekt bei der Erzeugung zu initialisieren.

- Konstruktor ist eine spezielle Methode einer Klasse, die automatisch aufgerufen wird, wenn ein Objekt der Klasse erzeugt wird.
- hat den gleichen Namen wie die Klasse und hat keinen Rückgabewert.

```cpp
class Vector2D {
...
public:
    // Konstruktur
    Vector2D() {
        x = 0;
        y = 0;
    }
    ...
};
```
- hier wurde ein s.g. *default-Konstruktor* definiert, der die Attribute `x` und `y` auf `0` setzt.

Ein Konstruktor kann auch Parameter haben, um das Objekt bei der Erzeugung zu initialisieren.
	z.B. macht es Sinn, zu erlauben, die Werte des Vektors gleich bei der Initialisierung des Objekts anzugeben:

```cpp
    ...
    // Konstruktor mit Parametern
    Vector2D(int x, int y) {
        this->x = x;
        this->y = y;
    }
```

### 3.4.1 Der 'this'-Zeiger
- ein spezieller Zeiger, der auf das aktuelle Objekt zeigt.
- wird implizit in jeder Methode einer Klasse definiert und kann genutzt werden, um auf die Attribute des Objekts zuzugreifen.

- Falls die Variablennamen in der Funktion eindeutig sind, kann der `this`-Zeiger auf weggelassen werden.
- Im obigen Fall haben die Parameter des Konstruktors die gleichen Namen wie die Attribute, daher muss man den `this` -Zeiger nutzen, um zu sagen, dass man die Attribute der Klasse verändern will.

### 3.4.2 Verwendung von Konstruktoren
Bei der Deklaration eines Objekts können wir nun den passenden Konstruktor aufrufen, um das Objekt zu initialisieren:

```cpp
int main() {
	Vector2D v1(3, 4);
	Vector2D v2(1, 2);
	...
}
```

### 3.4.3 Überladung
In `C++` können mehrere Funktionen gleichen Namens existieren, solange sie sich in der Anzahl oder den Typen der Parameter unterscheiden.
-> Dies nennt man *Überladung*.
- Überladung ist nicht nur für Konstruktoren möglich, sondern für alle Funktionen.

-> Häufig wird es aber bei Konstruktoren genutzt, um unterschiedliche Initialisierungen zu ermöglichen.
So könnten wir z.B. einerseits de default-konstruktor und andererseits eine Konstruktor mit Parametern anbieten:

```cpp
class Vector2D {
...
public:
    // default-Konstruktor
    Vector2D() {
        x = 0;
        y = 0;
    }

    // Konstruktor mit Parametern
    Vector2D(int x, int y) {
        this->x = x;
        this->y = y;
    }
    ...
};
```

### 3.4.4 Die Initialisierungsliste
Neben der Zuweisung der Attribute im Funktionsrumpf des Konstruktors können wir die Attribute auch als s.g. *Initialisierungsliste* direkt hinter der Deklaration des Konstruktors initialisieren.

Dazu schreibt man nach der Funktionsdeklaration einen Doppelpunkt `:` und dann die Initialisierung der Attribute in der Form `attribute(wert, …)`:

```cpp
class Vector2D {
...
public:
    // default-Konstruktor mit Initialisierungsliste
    Vector2D() : x(0), y(0) {}
    // Konstruktor mit Initialisierungsliste
    Vector2D(int x, int y) : x(x), y(y) {}
    ...
};
```
siehe [[4.3 Vererbung]] 

### 3.4.5 Der default-Konstruktor
- ein spezieller Konstruktor, der keine Parameter hat
- wird automatisch aufgerufen, wenn ein Objekt ohne Parameter erzeugt wird und hat zur Aufgabe, das Objekt auf sinnvolle Werte zu initialisieren.

- Falls man keinen Konstruktor explizit definiert, wird automatisch ein default-Konstruktor vom Compiler generiert.
	-> dieser ruft dann wiederum alle default-Konstruktoren der Member auf

- Sobald jedoch ein Konstruktor mit Parametern definiert wird, wird der implizite default-Konstruktor nicht mehr generiert.
- In diesem Fall müssen wir den default-Konstruktor explizit definieren.

Vergleich folgender Beispiele:

```cpp
class Vector2D {
private:
    int x;
    int y;
};

int main() {
    Vector2D v; // Kein Fehler, impliziter default-Konstruktor wird aufgerufen
    ...
}
```

```cpp
class Vector2D {
private:
    int x;
    int y;
public:
    Vector2D(int x, int y) : x(x), y(y) {}
};

int main() {
    Vector2D v; // Fehler, impliziter default-Konstruktor existiert nicht mehr, da wir einen Konstruktor definiert haben
    ...
}
```

> [!note]
> Es ist empfehlenswert, immer einen default-Konstruktor zu definieren, auch wenn er nur die Attribute auf `0` setzt.

### 3.4.6 Der Kopierkonstruktor
- wird automatisch aufgerufen, wenn ein Objekt durch Kopieren eines anderen Objekts erzeugt wird
	-> also z.B. bei der Übergabe eine Objekts an eine (*call-by-value-*)Funktion oder bei der Rückgabe eines Objekts aus einer Funktion.

Der Kopierkonstruktor hat als Parameter eine Referenz auf ein Objekt der gleichen Klasse:

```cpp
class Vector2D {
...
public:
    // Kopierkonstruktor
    Vector2D(const Vector2D& other) {
        x = other.x;
        y = other.y;
    }
    ...
};
}
```
- Im Kopierkonstruktor können wir implementieren, wie ein Objekt kopiert werden soll.
- In Obigem Beispiel kopieren wir einfach die Werte der Attribute des anderen Objekts.

- Definiert man keinen expliziten Kopierkonstruktor, wird ein impliziter Kopierkonstruktor vom Compiler generiert.
- Dieser kopiert einfach die Attribute des anderen Objekts.

## 3.5 Destruktoren
- *Konstruktor* zur Initialisierung eines Objekts und *Dekonstruktor* zum Löschen eines Objekts
- wird automatisch aufgerufen, wenn ein Objekt den Gültigkeitsbereich verlässt bzw. gelöscht wird.

- Der Destruktor hat den gleichen Namen wie die Klasse, wird jedoch mit einer Tilde `~` davor geschrieben und hat keinen Rückgabewert und keine Parameter:

```cpp
class Vector2D {
...
public:
    ...
    // (Expliziter) Destruktor
    ~Vector2D() {
        std::cout << "Vector2D-Objekt wird gelöscht" << std::endl;
    }
};
```

- wird kein Destruktor explizit definiert, wird ein impliziter Destruktor vom Compiler generiert.
	-> Dieser ruft die Destruktoren aller Member-Objekte auf.
	-> In vielen Fällen kommt man daher ohne expliziten Destruktor aus.
- Ausnahme:
	Wenn dynamisch allokierte Ressourcen (z.B. mit `new` ) innerhalb der Klasse verwendet werden. In diesem Fall wird der Destruktor benötigt, um die Ressourcen wieder freizugeben.

## 3.6 'const'-Methoden
- der Inhalt der Variable darf nicht verändert werden
- bei primitiven Datentypen (z.B. `int`, `double`, …) ist dies verständlich.

Aber was passiert bei Objekten?
- Haben wir ein Objekt als `const`, bedeutet dies, dass wir die Attribute des Objekts nicht verändern dürfen.
- Damit der Compiler weiß, dass wir in einer Methode keine Attribute verändern, müssen wir die Methode selbst als `const` deklarieren:

```cpp
class Vector2D {
...
public:
    ...
    void print() const {  //< Methode verändert keine Attribute -> const
        std::cout << "(" << x << ", " << y << ")" << std::endl;
    }
    ...
};
```
- Haben wir einen `const`-Vektor, können wir nun die Methode `print()` aufrufen, da sie als `const` deklariert ist.
-  Es ist auch möglich, zwei Versionen der gleichen Methode zu definieren, eine `const` und eine nicht-`const`. 
	-> Der Compiler wählt dann automatisch die passende Methode aus, je nachdem, ob das Objekt `const` ist oder nicht.

> [!note]
> Methoden, die keine Attribute verändern (d.h. nur _lesend_, nicht aber _schreibend_ auf die Attribute zugreifen), sollten immer als `const` deklariert werden!

## 3.7 Operatorüberladung
- man kann Operatoren für die eigene Klasse überladen
-> Das bedeutet, dass wir definieren können, was passieren soll, wenn ein Operator (z.B. *ist gleich* '\==', *kleiner* '<' oder *addiere* '+') auf ein Objekt unserer Klasse angewendet wird.

- Überladung erfolgt durch Definition einer speziellen Methode, deren Name durch das Schlüsselword `operator` und den Operator selbst angegeben wird. Zum Beispiel `operator+` für den `+`-Operator oder `operator==` für den '\==' -Operator.

Ein Beispiel für die Überladung des `+`-Operators und '\=='-Operators für unsere Klasse `Vector2D`:

```cpp
class Vector2D {
...
public:
    ...
    // Addition
    Vector2D operator+(const Vector2D& other) const {
        Vector2D result;
        result.x = x + other.x;
        result.y = y + other.y;
        return result;
    }

    // Gleichheit
    bool operator==(const Vector2D& other) const {
        return x == other.x && y == other.y;
    }
    ...
};

int main() {
    Vector2D v1(3, 4);
    Vector2D v2(1, 2);
    Vector2D v3 = v1 + v2;
    v3.print();

    if (v1 == v2) {
        std::cout << "v1 und v2 sind gleich" << std::endl;
    }

    ...
}
```
- Die Verwendung von Operatoren macht den Code leichter verständlich:
```cpp
Vector2D v3 = v1 + v2; //Intuitiv
```
- Ohne Parameterüberladung wäre der Code weniger elegant:
```cpp
Vector2D v3 = v1.add(v2); // Weniger intuitiv
```
